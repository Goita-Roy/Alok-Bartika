CLASS & OBJECT

🧠 সহজ ধারণা

১০০ জন প্লেয়ারের জন্য ১০০ বার আলাদা কোড লিখব না — একটাই নকশা বানাব!

এখন গেমের ১০০টি প্লেয়ারের জন্য কি তুমি ১০০ বার আলাদা আলাদা কোড লিখবে? একদমই না! তুমি বুদ্ধি করে প্রথমে প্লেয়ারের একটা কমন নকশা বা ডিজাইন বানিয়ে নেবে। প্রোগ্রামিংয়ের ভাষায় এই নকশাকেই বলে Class (ক্লাস)। আর এই নকশা দেখে দেখে যখন তুমি আসল এক একজন প্লেয়ার তৈরি করবে, তখন তাদের বলা হবে Object (অবজেক্ট)।

Class = Design (নকশা বা ব্লুপ্রিন্ট) 📐 Object = Real Player (আসল প্লেয়ার) 🎮

🎯 নির্দেশনা (Assignment)

চল এডিটর খুলে একটি নতুন game.py প্রোগ্রাম তৈরি করি।

STEP 01
📐

ডিজাইনের ছাঁচ

প্রথমে class কিওয়ার্ড ব্যবহার করে Player নামের একটি খালি ক্লাস বা নকশা তৈরি করো। ভেতরে কোনো কোড না থাকলে pass লিখে দেবে।

STEP 02
🧬

প্রথম প্লেয়ার তৈরি

ক্লাসের বাইরে এসে player1 = Player() লিখে তোমার গেমের প্রথম আসল প্লেয়ার বা অবজেক্টটি তৈরি করো।

STEP 03
🎮

তথ্য যোগ ও প্রিন্ট

ডট (.) চিহ্নের সাহায্যে প্রথম প্লেয়ারের নাম দাও "Alok" এবং লেভেল দাও 5। সবশেষে print() ব্যবহার করে প্লেয়ারের নাম আর লেভেল স্ক্রিনে দেখাও।

💡 সাহায্য, ভিডিও & সমাধান

💡 হিন্টস (Hints)▶

হিন্ট ১: পাইথনে ক্লাস তৈরি করার নিয়ম হলো:

class Player:
    pass

হিন্ট ২ (ডট ম্যাজিক): অবজেক্টের নামের পাশে ডট (.) দিয়ে সরাসরি ভ্যারিয়েবল বানিয়ে তথ্য বসিয়ে দেওয়া যায়! যেমন: player1.name = "Alok"

হিন্ট ৩: স্ক্রিনে প্রিন্ট করার সময় কমা (,) ব্যবহার করে অবজেক্টের ভেতরের তথ্যগুলো দেখাতে পারো, যেমন: print(player1.name)

🛠️ সমাধান ও সম্পূর্ণ কোড (Solution)▶

তোমার নতুন তৈরি করা game.py ফাইলের কোডটি হবে এরকম সহজ ও গোছানো:

game.py
# ১. প্লেয়ারের একটি খালি নকশা বা ক্লাস তৈরি করলাম class Player: pass # ২. প্রথম প্লেয়ারের অবজেক্ট (Real Player) তৈরি করলাম player1 = Player() # ৩. ডট (.) ব্যবহার করে প্লেয়ারের তথ্যগুলো বক্সে ভরে দিলাম player1.name = "Alok" player1.level = 5 print("--- 🎮 গেমের প্লেয়ার প্রোফাইল ---") # ৪. এবার অবজেক্টের ভেতরের তথ্য স্ক্রিনে প্রিন্ট করার পালা print("👤 প্লেয়ারের নাম:", player1.name) print("⭐ প্লেয়ারের লেভেল:", player1.level)
